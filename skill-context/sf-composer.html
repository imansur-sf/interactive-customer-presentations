Why Now

The tools marketers need already exist — what's been missing is the intelligence layer that connects them.
AI can reason about journeys
Claude Sonnet can ingest a stack configuration and generate a coherent, multi-step, multi-channel journey without any templates. The intelligence to design journeys is now available via API — not locked inside a vendor platform.
MCP makes every platform connectable
Model Context Protocol means Shopify, HubSpot, Segment, and Salesforce can all be orchestrated from a single AI session. The connector problem — the one that killed every previous MarTech unification attempt — is solved.
Journey complexity is at breaking point
Marketers now manage 7–10 platforms on average. Journey mapping takes weeks and goes stale before it's finished. The speed gap between strategy and execution is costing pipeline — every quarter.
Salesforce customers already have the data
Data Cloud customers have unified profiles ready to activate. SF Composer gives that data a fast path to execution — no new infrastructure, no new integration projects. The raw material is already there.

    Where This Goes
    Three extensions that open
    the full platform opportunity
    The canvas and AI layer are the foundation. These three extensions turn SF Composer from a demo asset into a product-grade journey design system.
    EXTENSION 01
    MuleSoft live execution bridge
    Every journey step with a MCP config path exports as a deployable MuleSoft flow. Journey design and integration design become the same artefact — built once, executed anywhere in the Anypoint platform.
    EXTENSION 02
    Data Cloud segment activation
    Journey entry conditions in the canvas map directly to Data Cloud segment definitions. When a customer's profile triggers the segment, the journey fires — no manual handoff between the design tool and the data platform.
    EXTENSION 03
    Multi-journey workspace
    Teams manage dozens of journeys simultaneously. A workspace layer lets marketers organise journeys by programme, campaign, or lifecycle stage — with shared stack configuration and reusable step templates across all of them.
